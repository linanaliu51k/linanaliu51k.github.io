          # session

          session的一些问题

          • 目前session直接是js变量，放在nodejs进程内存中
          • 第一，进程内存有限，访问量过大，内存暴增怎么办？
          • 第二，正式线上运行是多进程, 进程之间内存无法共享

          解决方案 redis

          • web server最常用的缓存数据库，数据存放在内存中
          • 相比于mysql,访问速度快(内存和硬盘不是一个数量级的)
          • 但是成本更高，可存储的数据量更小(内存的硬伤)
          • 将web server 和 redis拆分成为两个单独的服务
          • 双方都是独立的，都是可扩展的(例如都扩展成集群)
          • (包括mysql，也是一个单独的服务，也可扩展)

          为何session适合用redis？

          • session 访问频繁，对性能要求极高
          • session 可不考虑断电丢失数据的问题(内存的硬伤)
          • session 数据量不会太大(相比于mysql中存储的数据)

          为何网站数据不适合用redis ？

          • 操作频率不是太高(相比于session操作)
          • 断电不能丢失，必须保留
          • 数据量太大，内存成本太高

          # nginx 介绍

          • 高性能的web服务器,开源免费
          • 一般用于做静态服务、负载均衡
          • 还有反向代理
          // nginx 配置反向代理
location / {
  proxy_pass http://localhost:8001;
}
location /api/ {
  proxy_pass http://localhost:8000;
  proxy_set_header Host $host;
}
