                    # 层叠上下文

                    学习计划开始第五天，今天的目标是理解层叠上下文。

                    # 什么是层叠上下文

                    stacking context, 是HTML中一个三维的概念，如果一个元素含有层叠上下文,可以理解为这个元素在z轴上就”高人一等“。z轴，指的是用户与显示器之间一条看不见的垂直线。

                    我们可以把层叠上下文理解为一种“层叠结界”，自称一个小世界。可能包含其他层叠结界，也可能处于其他层叠结界中。

                    # 什么是层叠水平

                    stacking level, 决定了同一个层叠上下文中元素在z轴上的显示顺序。所有元素都有层叠水平，包括层叠上下文元素，也包括普通元素。层叠上下文本身是一个强力的层叠结界，普通元素的层叠水平是无法突破结界和结界外的元素去较量层叠水平的。

                    # 理解元素的层叠顺序

                    stacking order, 表示元素发生层叠时有着特定的垂直显示顺序。层叠上下文和层叠水平是概念，层叠顺序是规则.

                    css2.1时的层叠顺序规则如图

                    说明:

                    • 位于最下面的background/border特指层叠上下文元素的边框和背景 色。每一个层叠顺序规则仅适用于当前层叠上下文元素的小世界。
                    • inline水平盒子指的是包括inline/inline-block/inline-table元素 的“层叠顺序”，它们都是同等级别的
                    • 单纯从层叠水平上看，实际上z-index:0和z-index:auto是可以看成是一样的。实际上，两者在层叠上下文领域有着根本性的差异。

                    务必牢记的层叠准则, 是层叠领域的黄金准则。

                    • 谁大谁上: 当具有明显的层叠水平标识的时候，如生效的z-index属性值，在同一个层叠上下文领域，层叠水平值大的那一个覆盖小的那一个。
                    • 后来居上: 当元素的层叠水平一致、层叠顺序相同的时候，在DOM流中处于后面的元素会覆盖前面的元素。

                    # 深入理解层叠上下文

                    # 层叠上下文的特性

                    • 层叠上下文的层叠水平要比普通元素高
                    • 层叠上下文可以阻断元素的混合模式(没太查找相关规则)
                    • 层叠上下文可以嵌套，内部层叠上下文及其所有子元素均受制于外部的“层叠上下文”
                    • 每个层叠上下文和兄弟元素独立，也就是说，当进行层叠变化或渲染的时候， 只需要考虑后代元素。
                    • 每个层叠上下文是自成体系的，当元素发生层叠的时候，整个元素被认为是在 父层叠上下文的层叠顺序中。

                    # 层叠上下文的创建

                    和块状格式化上下文一样，层叠上下文也基本上是由一些特定的CSS属性创建的。 分为3个流派。

                    • 天生派: 页面根元素天生具有层叠上下文，称为根层叠上下文。可以看成是html元素,页面中所有元素一定处于至少一个层叠结界中.
                    • 正统派: z-index值为数值的定位元素的传统“层叠上下文”。

                    对于position值为relative/absolute以及Firefox/IE浏览器(不包括Chrome浏览器)下含有position:fixed声明的定位元素，当其z-index值不是auto的时候，会创建层叠上下文。chrome浏览器定位为fixed的时候，自动会创建层叠上下文，不用管z-index。

                    上面覆盖下面，父级是普通定位元素，里面子元素根据z-index值判断，谁大谁上

                    <div style="position:relative;z-index:auto;">
  <div style="position:absolute;z-index:2;width:20px;height:20px;background:red"></div>
</div>
<div style="position:relative;z-index:auto;">
  <div style="position:absolute;z-index:1;width:50px;height:50px;background:pink"></div>
</div>

                    下面覆盖上面，z-index为0会创建一个层叠上下文，会优先比较父级层叠上下文元素的层叠顺序。后来居上

                    <div style="position:relative;z-index:0;">
  <div style="position:absolute;z-index:2;width:20px;height:20px;background:red"></div>
</div>
<div style="position:relative;z-index:0;">
  <div style="position:absolute;z-index:1;width:50px;height:50px;background:pink"></div>
</div>
                    • 扩招派: 其他CSS3属性。
                    • 元素为flex布局元素(父元素display:flex|inline-flex)，同时z-index值不是auto。
                    • 元素的opacity值不是1。
                    • 元素的transform值不是none。
                    • 元素mix-blend-mode值不是normal。
                    • 元素的filter值不是none。
                    • 元素的isolation值是isolate。
                    • 元素的will-change属性值为上面2~6的任意一个(如will- change:opacity、will-chang:transform等)。
                    • 元素的-webkit-overflow-scrolling设为touch。

                    # 层叠上下文与层叠顺序

                    一旦普通元素具有了层叠上下文，其层叠顺序就会变高。分为两种情况:

                    • 如果层叠上下文元素不依赖z-index数值，则其层叠顺序是z-index:auto，可看成z:index:0级别;
                    • 如果层叠上下文元素依赖z-index数值，则其层叠顺序由z-index值决定。

                    元素一旦成为定位元素，其z-index就会自动生效，此时其z-index就是默认的auto，也就是0级别。会覆盖inline或block或float元素。而不支持z-index的层叠上下文元素天然是z-index:auto级别。也就意味着，层叠上下文元素和定位元素是一个层叠顺序的，于是当它们发生层叠的时候， 遵循的是“后来居上”准则。

                    后来居上示例, 谁在后面谁在上

                    <div style="transform:scale(1);width:50px;height:50px;background:pink;"></div>
<div style="position:absolute;top:0;width:100px;height:100px;background:red"></div>

                    <div style="position:absolute;width:100px;height:100px;background:red"></div>
<div style="transform:scale(1);width:50px;height:50px;background:pink;"></div>

                    # z-index负值

                    z-index虽然名为负数层级，但依然无法突破当前层叠上下文所包裹的小世界。z-index负值渲染的过程就是一个寻找第一个层叠上下文元素的过程，然后层叠顺序止步于这个层叠上下文元素。

                    实际应用:

                    • 可访问性隐藏。z-index负值可以隐藏元素
                    • IE8下的多背景模拟.
                    .box {
  background-image: url('1.png');
  position: relative;
  z-index: 0;
}
.box:before,
.box:after {
  content: '';
  position: absolute;
  z-index: -1;
}
.box:before {
  background-image: url('2.png');
}
.box:after {
  background-image: url('3.png');
}

                    <div class="box">我是一段纯文字...</div>

                    # z-index“不犯二”准则

                    对于非浮层元素，避免设置z-index值，z-index值没有任何道理需要超过2。原因如下:

                    • 定位元素一旦设置了z-index值，就从普通定位元素变成了层叠上下文元素，相互间的层叠顺序就发生了根本的变化。
                    • 避免z-index“一山比一山高”的样式混乱问题。

                    注意: 浮层元素等元素定位，弹框、出错提示、一些下拉效果等都不受这一准则限制。
