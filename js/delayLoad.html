                          # js延迟加载

                          2020-04-14 周二

                          学习计划开始第十一天，今天的目标是js延迟加载的方式总结。

                          # 什么是js延迟加载

                          js的延迟加载是指页面加载完毕后再加载js文件，有助于提升页面的加载速度。

                          load事件

                          当页面完全加载后(包括所有图像、js文件、css文件等外部资源),就会触发window上的load事件。图像也可以触发load事件。script元素也支持load事件。二者有点区别:

                          • 新图像不一定要从添加到文档后才开始下载，只要设置了src属性就会下载。注意事件绑定和src赋值顺序
                          • script元素只有在设置了src属性，并且添加到文档后，才会开始下载js文件。事件绑定和src赋值顺序不重要。

                          DOMContentLoaded事件

                          在形成完整的dom树之后就会触发，不理会图像，js文件，css文件或其他资源是否已下载完毕。支持在页面下载的早期添加事件处理程序。

                          # 延迟加载的方式

                          • defer属性（延迟脚本）

                          HTML4.01为script标签定义了defer属性。属性用途是标明脚本在执行时不会影响页面的构造。设置该属性相当于告诉浏览器立即下载，但延迟执行。

                          <!DOCTYPE html>
<html>
  <head>
      <script src="01.js" defer="defer"></script>
      <script src="02.js" defer="defer"></script>
  </head>
  <body>
  <!-- 这里放内容 -->
  </body>
</html>

                          虽然script元素放在了head元素中，但包含的脚本将在浏览器渲染到html结束标签后再执行。HTML5规范要求脚本按照它们出现的先后顺序执行。并且这两个脚本会先于DOMContentLoaded事件执行, 在现实当中，延迟脚本并不一定会按照顺序执行, 也不一定先于DOMContentLoaded事件。最好只有一个延迟脚本。

                          注意: 该属性只适用于外部文件。html5中已经明确规定。支持html5的实现会忽略给嵌入脚本(没有src属性)设置的defer属性。

                          • async属性（异步脚本）

                          HTML5为cript元素定义了async属性，与defer属性类似，都用于改变处理脚本的行为，只适用于外部脚本，并告诉浏览器立即下载文件。与defer不同的是，标记为async的脚本并不保证按照指定它们的顺序执行。

                          <!DOCTYPE html>
<html>
  <head>
      <script src="01.js" async></script>
      <script src="02.js" async></script>
  </head>
  <body>
  <!-- 这里放内容 -->
  </body>
</html>

                          该属性的目的是不让页面等待脚本下载和执行，从而异步加载页面其他内容。异步脚本不要在加载期间修改dom。异步脚本一定会在load事件前执行。但可能会在DOMContentLoaded事件触发之前或之后执行。

                          • 动态创建DOM方式
                          <script type="text/javascript">  
  function downloadJSAtOnload() {  
      varelement = document.createElement("script");  
      element.src = "defer.js";  
      document.body.appendChild(element);  
  }
  if (window.addEventListener)  
    window.addEventListener("load",downloadJSAtOnload, false);  
  else if (window.attachEvent)  
    window.attachEvent("onload",downloadJSAtOnload);  
  else
    window.onload =downloadJSAtOnload;  
</script>
                          • 使用jQuery的getScript()方法
                          $.getScript("outer.js",function() { //回调函数，成功获取文件后执行的函数  
  console.log("脚本加载完成")  
});
                          • 使用 setTimeout 延迟方法

                          对于不支持DOMContentLoaded的浏览器，可以使用该方法模拟。在当前js处理完成后立即调用这个函数，这个时间与DOMContentLoaded的触发时间不一定一致，主要取决于其他浏览器和页面中的其他代码，为了确保有效，必须作为页面中第一个超时调用。无法保证其调用一定早于load事件被触发。

                          setTimeout(function() {
// 在此添加事件处理程序
}, 0)
                          • 让js最后加载

                          讲js放在body闭合标签之前，确保页面结构和样式渲染完毕后再执行脚本文件。(常用)
